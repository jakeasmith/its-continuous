Its Continuous! v.2

The tweet-sized continuous integration app for PHP development.

What is continuous integration?

Continuous Integration is a software development practice where members of a team integrate their work frequently, usually each person integrates at least daily - leading to multiple integrations per day. Each integration is verified by an automated build (including test) to detect integration errors as quickly as possible.

Martin Fowler's article on Continuous Integration

Thanks to great packages like Composer, PHPUnit, Mockery and many others, writing tests for your PHP application is almost trivial. And if you're willing to invest a little bit of time there are some fantastic tools out there that will automatically run those tests for you when you push a new commit. Unfortunately, there hasn't been a tool that would constantly monitor your codebase as you work and report back the current status of your test suite in real time.

Until now. Welcome to Its Continuous. The little tweet-sized continuous integration app that will change your development practices in a big way.

What's it do?

Continuously running tests while you're writing code reveals bugs early. Unfortunately, the more tests you write, the longer it takes to run the entire test suite and the less likely you are to wait around for it. Its Continuous will run your tests for you so you don't have to think about it. Just start it up and leave it running; you'll see it flash red when tests start failing and have the satisfaction of watching them turn green as you bring things back into order.

Its immediate feedback that can only make things better. Test driven development has never been easier.

The Setup

Thanks to the suggestions of @Luke Carrier and Gigablah the original script has been rewritten. Instead of burning up CPU cycles to constantly run your tests it is now smart enough to only run tests when you have updated your code. Setup just requires a few easy steps.

  1. Make sure you install PHPUnit with composer.
  2. Use homebrew to install fswatch.
  3. Copy the app to your ~/.bash_profile.
  4. Restart your terminal and navigate to your project.
  5. Type tdd and watch the magic happen.

Your test suite will start running immediately and give you the latest results in the terminal. The great thing is that as soon as update your code the app immediately runs the tests again "in the background" and seamlessly updates the terminal without any distracting flickers.

First install fswatch with brew

brew install fswatch

Paste into ~/.bash_profile

# TDD CI
function tdd {
    clear; vendor/bin/phpunit
    fswatch ./src:./lib:./tests \
    'O=$(vendor/bin/phpunit); \
    clear; \
    echo "$O"'
}
